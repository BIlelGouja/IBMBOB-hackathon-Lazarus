## Executive Summary
- The Recode Live task proves that IBM Bob is used for actionable remediation, not only for static explanation.
- The backend asks IBM Bob to return `backendCode` and `reviewSections[].after`, which Lazarus uses for live replay and downloads.
- The prompt requires corrected code to be clean, with explanations kept in separate JSON fields.
- The UI can compare `reviewSections[].before` with `reviewSections[].after` to show a developer-friendly diff.
- The same structure supports single-file fixes, folder-level codebase analysis, and downloadable corrected artifacts.
